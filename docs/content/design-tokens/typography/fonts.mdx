# NCIDS Font Tokens

The NCIDS/USWDS provides a number of design tokens and utilities to deal with font styles:

1. [Font Family Tokens](#font-family-tokens) - These tokens set the family for the font. (e.g., `poppins`)
   - Set the font family using the `u-font-family` mixin or the `family` function.
     - `@include u-font-family( 'heading' )`
     - `font-family: family( 'heading' )`
2. [Font Size Tokens](#font-size-tokens) - These tokens set the size of the font. They must be used with a Font Family Token.
   - Set the font size using the `u-font-size` mixin or the `size` function.
     - `@include u-font-size( 'heading', 'lg' )`
     - `font-size: size( 'heading', 'lg' )`
   - Set the font family AND the font size using the `u-font` mixin.
     - `@include u-font-size( 'heading', 'lg' )`
3. [Font Weight Tokens](#font-weight-tokens) - These tokens set the weight of the font.
   - Set the font weight using the `u-text` mixin or the `font-weight` function.
     - `@include u-text( 'bold' )`
     - `font-weight: font-weight( 'bold' )`
4. [Line Height Tokens](#line-height-tokens) - These tokens set the line height for the font.
   - Set the line height using the `u-line-height` mixin or the `line-height` function
     - `@include u-line-height( 'heading', 2 )`
     - `line-height: line-height( 'heading', 2 )`
   - Set the family, size, and line height together using the `typeset` mixin.
     - `@include typeset( 'heading', 'lg', 2 )`

## Font Family Tokens

The NCIDS uses the same font family tokens from the USWDS and contains Type and role-based tokens.

- **Type-based tokens** are based on the [available fonts](#available-fonts), e.g. `poppins` or `open-sans`.
- **Role-based tokens** use the type-based tokens and are based on the role the text will play. For example, button and other input text should use the `ui` role; web page content should use the `body` role.
  - Role-based tokens should be used whenever possible. The idea is to group like elements so that if the font family is changed, all those elements of that role change. This is opposed to wanting to change all `heading` items, but you need to find everywhere in the code that something is considered a header and then change all of those places. (Experience shows some places will be missed.)

## Font Size Tokens

There are two types of size tokens, `system` and `theme` tokens.

- **Theme Tokens** - These are a set of 9 common sizes based on the various designs and should be preferred over using the system sizes directly.
- **System Tokens** - These are a set of sizes numbered from 1 - 20, that correspond to a pixel size. These should only be used in rare circumstances if you cannot use a theme token.

> NOTE: The USWDS uses a [normalization system](https://designsystem.digital.gov/design-tokens/typesetting/overview/) to ensure fonts at a certain size are optically similar. The pixel sizes in the tables below are showing the target for the size. The actual font-size will vary. (For example, Open Sans @ 28px is really 28.32px)
