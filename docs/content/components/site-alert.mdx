# Site alert

A site alert communicates urgent sitewide information.

## Site alert guidance

- Use the blue site alert for the COVID content.
- Use the red site alert for government shutdowns.
- When there is more than one link, the site alert becomes a drop down that can be opened and closed to reveal links in a bulleted list (separated by columns for desktop).
- Users need the opportunity to close a site alert so that it no longer appears during their time on the site. This should be an optional setting in drupal. Ex: gov shutdown site alert would not get the "x".
- Site alerts include the icons from USDWS.
- Use the COVID alert according to NCI's policy. Do not change icons, text, colors, links, or functionality.
- Use the Shutdown alert according to NCI's policy. Do not change icons, text, colors, links, or functionality.

### When to use the site alert component

- **Critical system notifications.** Use a site alert to deliver a static system status update, such as notices about unavailable services or content. Site alerts should appear by default and not in response to an action.
- **Messages that should be displayed on every page.** A site alert makes critical information obvious and findable on every page.

### When to use something else

- **Responding to user actions.** Use the alert component for page-level validation messages (like error messages on a form) or to inform a user that a task was completed successfully.
- **Validating form input.** On most forms, especially longer forms, validation messages should appear inline with error messages at the top of the form.

### Usability guidance

- **Place prominently.** Use a full-width site alert positioned near the top of the page so it’s one of the first things users see.
- **Avoid stacking multiple site alerts.** If you need to convey more than one message, provide a list of links within a single site alert instead of multiple, stacked alerts.
- **Avoid overwhelming use of color.** It may be tempting to make heavy use of bright red or orange to draw attention to an emergency-related site alert. However, these colors can produce strong negative emotional reactions such as fear or panic. Positioning the banner at the top of the screen provides enough weight that such strong colors may not be necessary.

### Accessibility

- **Don’t visually hide alert messages and then make them visible when they are needed.** Users of older assistive technologies may still be able to perceive hidden alert messages. Fully remove alert messages when they’re not needed.
- `aria-label` must be unique to the page and should describe the purpose of the alert.

**_Aria-Label Translations Table_**

| **English label**      | **Spanish label**             |
| ---------------------- | ----------------------------- |
| `Dismiss Alert`        | `Cerrar notificación`         |
| `Toggle alert message` | `Abrir o cerrar notificación` |

### Usage

#### Options

```javascript nopreview
// Defaults
{
	closeable: false,
	closeButton: {
		ariaLabel: 'Dismiss alert',
		buttonClass: 'usa-alert__nci-button--close',
		eventListenerLabel: 'usa-site-alert:close-button',
		cookiePath: '/'
	},
	collapse: {
		ariaLabel: 'Toggle alert message',
		buttonClass: 'usa-alert__nci-button--toggle',
		eventListenerLabel: 'usa-site-alert:content',
}
```

- `{boolean}` **`closeable`** - Add close button to component
- **`closeButton`**
  - `{string}` **`ariaLabel`** - Unique label used to describe the close button
  - `{string}` **`buttonClass`** - Button class that will close the component
  - `{string}` **`cookiPath`** - Cookie path to set
- **`collapse`**
  - `{string}` **`ariaLabel`** - Unique label used to describe the button
  - `{string}` **`buttonClass`** - Button class that toggles collapsible section
  - `{string}` **`eventListenerLabel`** - Label used in analytics

**Note:** Site alerts require a unique ID to enable both collapsible and dismissible sections.

### Dismissible site alert

Add a close button to any site alert by setting the `closable` property to `true`.

```javascript nopreview
NCISiteAlert.create(element, { closeable: true });
```

## Package

Import one of the following Sass partials into your stylesheet

```scss nopreview
@forward "usa-site-alert";
```

<NciDsJsInit>{`
	const elements = preview.querySelectorAll('.usa-site-alert');
	elements.forEach((element) => {
    const label = element.getAttribute('aria-label');
    switch (label) {
        case 'Standard emergency site alert example':
            window.ncids.NCISiteAlert.create(element, { collapse: {ariaLabel: 'Toggle standard emergency alert example'} });
            break;
        case 'Standard information site alert example':
            window.ncids.NCISiteAlert.create(element, { collapse: {ariaLabel: 'Toggle standard info alert example'} });
            break;
        case 'Slim information site alert example with close':
            window.ncids.NCISiteAlert.create(element, {closeable: true, closeButton: {ariaLabel: 'Dismiss slim alert example'} });
            break;
        case 'Standard information site alert example with close':
            window.ncids.NCISiteAlert.create(element, {closeable: true, closeButton: {ariaLabel: 'Dismiss standard alert example'}, collapse: {ariaLabel: 'Toggle standard info alert with close example'}});
            break;
    }
});
`}</NciDsJsInit>
